# 從零開始操作教學

## 步驟 1：建立 Supabase 專案

1. 前往 https://supabase.com/ 並登入。
2. 按 `New project`。
3. 輸入專案名稱，例如 `nrs-pain-assessment`。
4. 設定並妥善保存 Database Password。
5. 選擇離使用者較近的 Region。
6. 按 `Create new project`，等待專案建立完成。

## 步驟 2：建立資料庫

1. 在 Supabase 左側選單按 `SQL Editor`。
2. 按 `New query`。
3. 打開本專案的 `supabase/setup.sql`。
4. 全選並複製所有內容。
5. 貼到 Supabase SQL Editor。
6. 按 `Run`。
7. 顯示成功後，在 `Table Editor` 應看見：
   - `profiles`
   - `pain_assessments`

`setup.sql` 僅供空白專案第一次執行。如果已執行成功，不要重複執行。

## 步驟 3：建立兩個測試帳號

1. 左側選單按 `Authentication`。
2. 開啟 `Users`。
3. 按 `Add user` 或 `Create new user`。
4. 建立一般使用者：
   - Email：`user@example.com`
   - Password：自行設定至少 8 碼密碼
   - 勾選自動確認使用者（Auto Confirm User）
5. 再建立管理員：
   - Email：`admin@example.com`
   - Password：自行設定至少 8 碼密碼
   - 勾選自動確認使用者

建立帳號後，SQL 內的 Trigger 會自動在 `profiles` 表建立 `user` 角色。

## 步驟 4：把管理員改成 admin

1. 到 `Authentication → Users`。
2. 找到 `admin@example.com`。
3. 複製該帳號的 UUID。
4. 回到 `SQL Editor → New query`。
5. 貼上以下 SQL，把 UUID 替換成剛才複製的值：

```sql
update public.profiles
set role = 'admin'
where id = '57eff666-e62c-4d82-90e8-b7bcfb2c6e6f';
```

6. 按 `Run`。
7. 到 `Table Editor → profiles`，確認管理員那一列的 `role` 是 `admin`。

## 步驟 5：取得 Supabase 連線資料

1. 在 Supabase 專案中開啟 `Project Settings`。
2. 找到 `API` 或 `Data API`。
3. 複製：
   - Project URL
   - Publishable key（部分舊專案顯示 anon key）
4. 打開 `js/config.js`。
5. 只替換引號中的兩個值：

```javascript
const SUPABASE_URL = "https://mnpridddwzrudwxkmelc.supabase.co";
const SUPABASE_PUBLISHABLE_KEY = "sb_publishable_9_cReVcyKcfCTRLJDReE6w_irS3c-nd";
```

不要把 `service_role` 或 secret key 放進網頁。

## 步驟 6：啟動網站

請不要直接雙擊 HTML。用終端機進入本專案資料夾：

```bash
cd "/Users/corrinexie/Desktop/護理資訊教學/nrs-supabase-mvp"
python3 -m http.server 5500
```

終端機保持開啟，再用瀏覽器前往：

```text
http://localhost:5500
```

停止伺服器時，在終端機按 `Control + C`。

## 步驟 7：測試一般使用者

1. 使用 `user@example.com` 登入。
2. 填寫一筆 NRS 疼痛評估。
3. 按「儲存評估」。
4. 前往「我的紀錄」。
5. 應看到剛才新增的資料。
6. 一般使用者開啟 `admin.html` 時，應被導回登入頁。

## 步驟 8：測試管理員

1. 先登出一般使用者。
2. 使用 `admin@example.com` 登入。
3. 系統會自動進入管理者頁面。
4. 管理員應能看到所有使用者的評估紀錄。

## 常見問題

### 顯示「尚未設定 Supabase」

代表 `js/config.js` 還沒有貼入正確的 Project URL 和 publishable key。

### 登入失敗

確認：

- Email 與密碼正確。
- 建立帳號時已選 Auto Confirm User。
- Supabase 專案沒有暫停。

### 儲存時出現 row-level security 錯誤

確認：

- 已完整執行 `supabase/setup.sql`。
- 使用者已登入。
- `pain_assessments` 表有 `user_id` 欄位。

### 管理員看不到全部資料

到 `Table Editor → profiles`，確認管理員帳號 UUID 對應的 `role` 是 `admin`。
